POST/api/donations

Buat donasi baru. Generate QRIS dinamis dengan nominal = amount + unique_code. Disimpan ke database dengan status pending.

Rate limit: 10 request per 5 menit per IP.

Body (JSON)

{
  "amount": 10000,           // Wajib, integer. Min: min_amount, Max: max_amount - 200
  "donorName": "John Doe",   // Opsional, max 100 karakter
  "message": "Semangat!"     // Opsional, max 500 karakter
}

Response 200

{
  "ok": true,
  "donationId": 1,
  "qrToken": "abc123def456...",
  "baseAmount": 10000,
  "baseAmountFormatted": "Rp 10.000",
  "uniqueCode": 23,
  "amount": 10023,
  "amountFormatted": "Rp 10.023",
  "qrImage": "data:image/png;base64,...",
  "dynamicQris": "00020101021226...",
  "expiresAt": "2026-06-18T08:00:00.000Z",
  "message": "Bayar tepat Rp 10.023 agar donasi terdeteksi otomatis."
}

💡 uniqueCode adalah angka acak 1–200 yang ditambahkan ke nominal agar setiap transaksi unik dan bisa dideteksi otomatis via notifikasi e-wallet.

Error

  • 400 — Nominal tidak valid / di luar batas min–max
  • 429 — Rate limit (10 req / 5 menit)
  • 500 — QRIS static belum dikonfigurasi di admin settings

POST/api/donations/:token/paid 🔒

Konfirmasi donasi secara manual by token. Dilindungi WEBHOOK_SECRET. Set status ke paid dan trigger webhook notifikasi.

Secret wajib dikirim via header atau body — bukan query string (query string muncul di access log).

Header (direkomendasikan)

X-Webhook-Secret: YOUR_WEBHOOK_SECRET

Atau via Body JSON

{ "secret": "YOUR_WEBHOOK_SECRET" }

cURL

curl -X POST "http://localhost:3003/api/donations/abc123/paid" \
  -H "X-Webhook-Secret: YOUR_WEBHOOK_SECRET"

Response

{ "ok": true, "message": "Donasi dikonfirmasi", "donation": { ... } }

Error

  • 401 — Secret salah atau tidak ada
  • 404 — Token tidak ditemukan
  • 500 — WEBHOOK_SECRET belum diset di server

🤖 Macrodroid Auto-Confirm

Macrodroid menangkap notifikasi e-wallet (GoPay, DANA, OVO, dll.) dan POST ke endpoint ini. Backend parse nominal "Rp X.XXX" dari teks notifikasi → cocokkan ke donasi pending → mark paid.

Auth: Basic Auth. Set MACRODROID_USERNAME dan MACRODROID_PASSWORD di .env. Header: Authorization: Basic base64(user:pass)

POST/api/macrodroid/confirm 🔒 Basic Auth

Terima notifikasi e-wallet dari Macrodroid. Parse nominal dari teks notifikasi → cari donasi pending yang cocok (dibuat < 2 jam lalu) → mark paid → trigger webhook.

Rate limit: 30 request per menit.

Cara kirim data (salah satu atau kombinasi)

  • JSON body (direkomendasikan) — Content-Type: application/json
  • URL query string — parameter di-append ke URL
  • Form-urlencoded — Content-Type: application/x-www-form-urlencoded

Body JSON (format direkomendasikan)

{
  "query": {
    "notif_text":  "Kamu telah menerima pembayaran Rp 10.023 dari xxx atas nama DIANXXXX.",
    "notif_app":   "com.dana",
    "notif_title": "Payment received",
    "status":      "paid",
    "timestamp":   1756476503893
  }
}

Field yang diterima

FieldAliasKeterangan
notif_text notifText, text, message, body, content Teks notifikasi e-wallet. Wajib mengandung pola Rp X.XXX atau IDR X.XXX
notif_app notifApp, app, app_name, packageName Package name app, e.g. com.dana, com.gojek.app
notif_title notifTitle, title Judul notifikasi (opsional)
status notification_status Jika diisi dan bukan paid / received / success, request diabaikan
timestamp time Unix ms timestamp (opsional)

cURL

curl -X POST "http://localhost:3003/api/macrodroid/confirm" \
  -H "Authorization: Basic $(echo -n 'macrodroid:PASSWORD' | base64)" \
  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "notif_text": "Kamu telah menerima pembayaran Rp 10.023 dari xxx.",
      "notif_app":  "com.dana",
      "status":     "paid"
    }
  }'

Response sukses

{
  "ok": true,
  "message": "Donasi dikonfirmasi via Macrodroid",
  "donation": {
    "id": 1,
    "baseAmount": 10000,
    "baseAmountFormatted": "Rp 10.000",
    "uniqueCode": 23,
    "amount": 10023,
    "amountFormatted": "Rp 10.023",
    "donorName": "John Doe",
    "message": "Semangat!",
    "status": "paid",
    "paidAt": "2026-06-17T08:05:00.000Z",
    "paidVia": "macrodroid",
    "paidViaApp": "com.dana",
    "matchedFrom": "Kamu telah menerima pembayaran Rp 10.023 dari xxx."
  }
}

Error

  • 400 — Gagal parse nominal dari notif_text (hint disertakan di response)
  • 401 — Basic Auth salah atau tidak ada
  • 404 — Tidak ada donasi pending dengan nominal tersebut, atau sudah lebih dari 2 jam
  • 429 — Rate limit (30 req / menit)

🔒 Admin Auth

POST/api/admin/login

Login admin. Set session cookie drp.sid. Default: admin / admin123 — wajib diganti setelah setup pertama.

Body

{ "username": "admin", "password": "admin123" }

Response

{ "ok": true, "username": "admin" }

cURL (simpan cookie ke file)

curl -X POST http://localhost:3003/api/admin/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"admin","password":"admin123"}'

POST/api/admin/logout

Hapus session. Tidak butuh body.

Response

{ "ok": true }

GET/api/admin/me

Cek apakah session aktif dan ambil info admin saat ini.

Response (sudah login)

{ "ok": true, "username": "admin", "loginAt": "2026-06-17T08:00:00.000Z" }

Response (belum login)

{ "error": "Not logged in" }  // HTTP 401

🎨 Branding

POST/api/admin/branding/logo 🔒

Upload logo kreator. Disimpan ke /uploads/logos/ dan otomatis update setting avatar_url.

Maks 2MB. Format: JPG, PNG, GIF, WEBP. (SVG tidak didukung karena risiko XSS.)

Form Data

field: file  (binary image)

cURL

curl -b cookies.txt -X POST http://localhost:3003/api/admin/branding/logo \
  -F "file=@logo.png"

Response

{ "ok": true, "url": "/uploads/logos/1718611200000-abc123.png", "filename": "1718611200000-abc123.png" }

POST/api/admin/branding/banner 🔒

Upload banner halaman sawer. Disimpan ke /uploads/banners/ dan otomatis update setting banner_url.

Maks 2MB. Format: JPG, PNG, GIF, WEBP.

cURL

curl -b cookies.txt -X POST http://localhost:3003/api/admin/branding/banner \
  -F "file=@banner.jpg"

Response

{ "ok": true, "url": "/uploads/banners/1718611200000-xyz789.jpg", "filename": "1718611200000-xyz789.jpg" }

DELETE/api/admin/branding/:type 🔒

Reset branding ke default. :type = logo (reset ke /images/logo_blue.webp) atau banner (kosongkan).

Response

{ "ok": true }

🔔 Webhooks

GET/api/admin/webhooks 🔒

List semua webhook yang terdaftar.

POST/api/admin/webhooks 🔒

Tambah webhook baru.

Body

{
  "name":       "Discord Notif",
  "type":       "discord",          // "discord" | "telegram" | "custom"
  "url":        "https://discord.com/api/webhooks/...",
  "enabled":    true,
  "trigger_on": "paid",             // "paid" | "created" | "both"
  "secret":     "optional-secret"  // opsional, dikirim sebagai X-Webhook-Secret header
}

Response

Object webhook yang baru dibuat (termasuk id).

PUT/api/admin/webhooks/:id 🔒

Update webhook. Body partial — hanya sertakan field yang ingin diubah.

DELETE/api/admin/webhooks/:id 🔒

Hapus webhook permanen.

Response

{ "ok": true }

POST/api/admin/webhooks/:id/test 🔒

Kirim payload dummy ke webhook untuk verifikasi koneksi.

Response

{ "ok": true, "status": 200, "duration": 120, "response": "ok" }

GET/api/admin/webhook-logs 🔒

Lihat log pengiriman webhook, join dengan nama dan tipe webhook.

Query Params

  • limit — 1–200 (default: 50)

💰 Admin Donations

GET/api/admin/donations 🔒

List semua donasi dengan filter dan pagination.

Query Params

  • page — default 1
  • limit — default 20, maks 100
  • status — pending | paid | expired | cancelled
  • period — today | month
  • q — search by nama donor, pesan, atau token

Response

{
  "page": 1,
  "limit": 20,
  "total": 150,
  "totalAmount": 7500000,
  "items": [ { "id": 1, "qr_token": "...", "amount": 10023, "status": "paid", ... } ]
}

GET/api/admin/donations/:id 🔒

Detail satu donasi by ID numerik.

Error

  • 404 — Donasi tidak ditemukan

PATCH/api/admin/donations/:id/status 🔒

Update status donasi secara manual. Jika berubah menjadi paid, webhook notifikasi dijalankan.

Body

{ "status": "paid" }   // "pending" | "paid" | "expired" | "cancelled"

Response

Object donation yang sudah diupdate.

DELETE/api/admin/donations/:id 🔒

Hapus donasi permanen.

Response

{ "ok": true }

GET/api/admin/donations/export.csv 🔒

Download semua donasi sebagai CSV. UTF-8 dengan BOM (siap dibuka di Microsoft Excel).

Kolom: ID, Token, Amount, Donor Name, Message, Status, Paid Via, Created At, Paid At

cURL

curl -b cookies.txt "http://localhost:3003/api/admin/donations/export.csv" -o donations.csv

🔍 QRIS Preview

POST/api/admin/qris/preview 🔒

Upload gambar QRIS statis untuk auto-decode string QRIS-nya, plus preview QR dinamis dengan nominal Rp 10.000 untuk verifikasi.

File temporary langsung dihapus setelah di-decode. Maks 2MB. Format: JPG, PNG, GIF, WEBP.

Form Data

field: file  (binary image QRIS)

cURL

curl -b cookies.txt -X POST http://localhost:3003/api/admin/qris/preview \
  -F "file=@qris.png"

Response

{
  "ok": true,
  "qris": "00020101021126...",
  "previewQr": "data:image/png;base64,..."
}

Error

  • 400 — Tidak ada file yang diupload
  • 500 — Gagal decode QR dari gambar
